# Ulobby — Brand Voice Reference

**Date:** 2026-06-08
**Source:** Copy analysis across homepage, about/story, solutions, plans, and blog.

---

## Tone Adjectives

1. **Authoritative** — speaks from deep domain knowledge of Public Affairs. Uses insider framing ("we see teams struggle with...") without hedging. Positions Ulobby as the expert who understands the discipline, not just the technology.
2. **Measured** — avoids hype, superlatives, and urgency. Sentences are calm and deliberate. No exclamation marks. No "revolutionary" or "game-changing." The confidence is quiet.
3. **Structured** — the copy itself models the product's value proposition. Ideas are broken into clear pillars, parallel constructions, and diagnostic frameworks. Copy reads like a well-organised brief.
4. **Pragmatic** — focuses on real workflow problems (lost context, fragmented tools, knowledge walking out the door) rather than aspirational outcomes. Grounded in the daily reality of PA work.
5. **European-professional** — British English spelling, formal-but-not-stiff register. No Americanisms, no startup vernacular. Reads like a Copenhagen-Brussels corridor voice: direct, efficient, internationally minded.

---

## Vocabulary Level

**Professional-specialist.** Roughly Flesch 55-65. Sentences are clear but assume the reader already knows what Public Affairs is, what stakeholder management means, and why political monitoring matters. No hand-holding explanations of domain concepts. Technical terms (PA, stakeholder mapping, issue monitoring) are used without definition because the audience lives inside them daily.

## Do Words

| Word/Phrase | Usage pattern |
|---|---|
| Public Affairs | Always capitalised. Always the full phrase, never abbreviated to "PA" in customer-facing copy. |
| structure / structured | Core value word. Appears across every page. "Structured overview", "structured reporting", "work with structure." |
| overview | Preferred to "dashboard" or "view." Implies strategic comprehension, not just data display. |
| continuity | Key differentiator concept. The problem is things getting lost; the solution is continuity. |
| context | Paired with continuity. "Context and follow-ups don't get lost." |
| over time | Temporal framing appears frequently. PA work unfolds over months and years. |
| teams | Primary user unit. Not "users" or "professionals" in isolation — always team-oriented. |
| one platform / one place / one shared system | Consolidation messaging. Repeats the "fragmented to unified" narrative. |
| visibility / visible / oversight | What the product creates. "Make efforts visible", "keep oversight." |
| follow-ups | Specific PA workflow term. Used instead of generic "tasks" or "to-dos." |
| positions | Stakeholder positions on issues. Domain-specific and used precisely. |
| manage | Neutral, professional verb. "Manage your work", "manage issues." Never "supercharge" or "turbocharge." |
| professionalize | Mission word. Used on the about page as the founding intent. |
| discipline | Public Affairs as a discipline, not a function or department. Elevating language. |

## Don't Words

| Avoided word/concept | What they use instead |
|---|---|
| Revolutionary / game-changing / cutting-edge | No hype language anywhere. Product is described by what it does, not what it disrupts. |
| AI-powered (as a headline) | AI is mentioned but buried. Never leads the messaging. Structure and workflow lead. |
| Simple / easy / effortless | Not used. The domain is complex; the product brings structure to complexity, not false simplicity. |
| Users | "Teams", "organisations", "professionals." |
| Dashboard | "Overview." |
| Real-time | Not used. The temporal frame is "over time", not instant. |
| Leverage / utilize / synergy | No consulting jargon. Plain European English. |
| Free / discount / limited time | No promotional urgency language. All plans require a demo request. |
| Hack / trick / secret | No clickbait patterns, even in blog content. |
| Best / leading / #1 | No superlative positioning claims. |
| Tool (as product descriptor) | "Platform" or "software." "Tool" is used only for generic concepts like "stakeholder mapping is a tool." |
| Excited / thrilled / passionate | No emotional startup language. |

## Sentence Style

- **Medium-length sentences.** Typical range: 12-22 words. Longer than startup copy, shorter than academic writing.
- **Declarative leads.** Statements of fact or diagnosis open most sections. No questions in headlines.
- **Problem-then-structure pattern.** Names a workflow problem, then shows how structure resolves it.
- **Parallel constructions.** Feature pillars use consistent syntax: "Follow political developments", "Keep a structured overview", "Turn activity into reporting."
- **Third-person authority.** Blog uses "we see" and "teams" rather than "you" direct address.
- **No imperative CTAs.** CTAs are invitations: "Get a short walkthrough", "See the platform in action." Never "Start now!"
- **Subordinate clauses earn their place.** Complex sentences exist because the idea requires qualification, not because the writing is loose.

## Voice Consistency Notes

**Strengths:**
- Remarkably consistent register across all pages. The measured, structured tone holds from homepage through blog through pricing.
- Blog voice and product voice are the same person. No jarring shift.
- Avoidance of hype language is genuine and sustained. Builds credibility with a senior, sceptical audience.
- "Structure over activity" narrative runs through every page — coherent philosophy, not just a feature list.

**Weaknesses / Inconsistencies:**
- Plans page is noticeably thinner. Tier descriptions are generic and could describe any SaaS product.
- "AI-powered" appears in some CTAs and meta descriptions but is not integrated into the brand voice. Reads like an SEO addition bolted onto otherwise AI-agnostic copy.
- CTAs vary without clear logic: "Get started", "See how it works", "Get a short walkthrough", "Request a demo" all appear to lead to the same action. Dilutes the conversion path.
- Homepage feature descriptions trend toward the generic; blog tells you why, homepage only tells you what.
