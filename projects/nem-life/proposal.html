  • A previous developer started the Webflow build but the engagement didn't work out.

  • The NL locale is configured but disabled.

Solution

  • Complete the build across 5 focused phases: build the remaining templates, make every page fully responsive, wire all navigation and plumbing, then launch with a solid technical foundation.
  • Includes 301 redirects from the existing nemlife.com domain, structured data schema, and a full accessibility pass.
  • Daily Slack check-ins and Loom walkthroughs throughout — no surprises.

Value Provided

  • Launch-ready NL website — every brief item delivered.
  • AI-search-friendly: Article, Organization, FAQPage and Person schema — positioning NEM Life for visibility in ChatGPT, Perplexity and Google AI Overviews.
  • Clean plumbing: every link works, NL locale live, EN locale parked and ready for Phase B activation.
  • Accessible & performant: axe-core tested, keyboard navigable, reduced-motion safe, Lighthouse optimised.

Project plan & scope

Phase 1 — Blog Insights Item Page (new build)

  • Sticky side panel: Share & Care, Kerninzichten, Essentie quote, Zelftest CTA card, Gerelateerde inzichten.
  • Modular body: H2/paragraph/image/quote/mid-page CTA via Rich Text.
  • "De essentie" checklist.
  • Full "Zet de volgende stap" 3-card CTA.
  • Desktop + mobile.

Phase 2 — Experiences Item Page (new build)

  • Single-column text-only template.
  • H1 + intro + paragraph body.

Phase 3 — Mobile Responsive (all pages)

  • Home (effective rebuild from design)
  • NEM Methode content page
  • Blog Insights main / category / item
  • Experiences main / item
  • Our Mission / Over Ons
  • Link-in-Bio
  • Blank page (T&C / Privacy / Cookie)
  • Nav + notification bar stacking, typography scale, image sets

Phase 4 — Debug, Plumbing & Interactions

  • Wire all nav + footer links.
  • Enable NL locale, hide EN.
  • Home: 3-image hover sequence, 123 card fade-in, FAQ single-open.
  • Blog Insights: sticky category bar.
  • Experiences: journey-type filter, dynamic CTA, load-more.
  • Shrink-on-scroll nav. Notification bar per-page show/hide.
  • Console-clean pass.

Phase 5 — Performance, SEO, AEO, Accessibility & Launch

  • Preload hero, WebP conversion, responsive image sets.
  • Meta + OG + Article/Organization/FAQPage/Person schema.
  • robots.txt, sitemap, hreflang.
  • AEO DOM audit — essential text in DOM at load.
  • axe-core pass on every template. Keyboard nav + reduced-motion.
  • Lighthouse baseline + targeted fixes.
  • 301 map from TEMP site, DNS cutover, final publish, smoke test.
  • Loom handover.

Content (text, photos and videos) must be provided and checked by the client. The site currently contains placeholder content — templates will be built against this and a content-pass revision is included.
