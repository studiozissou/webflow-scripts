### Two ways of working

Both options deliver the same result. The difference is sequencing.

#### Option A: Waterfall

All design is completed and signed off before any development begins.

How it works:

1. Full site design (all 12 pages) completed in Figma.
2. TSC reviews and signs off the complete design.
3. Development begins and proceeds page by page.

Pros:

- You see the entire site design before any build work starts.
- Easier to keep visual consistency across all pages.
- Clear handoff point between design and development.

Cons:

- Slower overall. Development cannot start until all designs are approved.
- Content for all pages must be ready before design begins.

#### Option B: Page-by-page (recommended)

Design and development run in parallel, staggered by page.

How it works:

1. Once a page design is complete and signed off, it goes straight into development.
2. While that page is being built, the next page is being designed.
3. This continues until all pages are done.

Pros:

- Faster overall. Design and development overlap.
- Content can be delivered page by page, which gives your team more flexibility.
- You see working pages sooner, which helps catch issues early.

Cons:

- Requires disciplined content delivery. If a page's content is late, the pipeline stalls.
- Design changes to earlier pages (after seeing later ones) may require rework.

### Process

In Waterfall, the steps below happen sequentially for all pages. In Page-by-page, steps 1 and 2 happen once upfront, then steps 3–5 repeat per page, so while one page is in development, the next is being designed.

1. Branding and UI direction. I design 3 options based on brand guidelines. TSC picks one and gives feedback.
2. Wireframes. Desktop wireframes of all pages. One round of feedback.
3. Desktop and mobile design. Full design for the page(s). Feedback gets factored into development.
4. Desktop build. I build the page(s) and provide a test version for review.
5. Responsive build. Responsive version for review. Final changes made here.
6. De-snagging. Last corrections, plus training and documentation on how to use the site.
7. Site launch and handover to TSC's Webflow account.
